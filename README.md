# M&A Analyst Copilot

## Overview

M&A Analyst Copilot is a Python-based workflow designed to assist junior investment banking analysts in reviewing public transaction documents.

The tool is designed to reduce the time spent manually reviewing:

- HKEX announcements
- Annual reports
- Investor presentations
- Press releases

Instead of replacing analyst judgement, it prepares a structured first-pass briefing covering:

- Transaction overview
- Strategic rationale
- Financing structure
- Synergies
- Risks
- Regulatory considerations
- Questions for management

This project is being developed as an ongoing prototype inspired by front-office M&A workflows.

## Business Problem

Junior M&A analysts spend significant time manually reviewing transaction announcements, annual reports, investor presentations and press releases before beginning financial analysis.

This repetitive process often involves extracting the same categories of information across every transaction, including:

- Transaction overview
- Strategic rationale
- Financing structure
- Expected synergies
- Key risks
- Regulatory approvals
- Potential due diligence questions

The objective of this project is not to replace analyst judgement.

Instead, it explores how AI can automate repetitive document review, allowing analysts to dedicate more time to valuation, client work and investment decision-making.

## Vision

This project aims to prototype an internal "M&A Analyst Copilot" inspired by the increasing adoption of AI tools across financial institutions.

Rather than generating investment recommendations, the tool prepares structured analyst briefing notes from publicly available transaction documents.

Future versions will explore integration with valuation workflows, comparable transactions, due diligence preparation and presentation materials.

## Analyst Workflow

Current Workflow

1. Read transaction announcement
2. Read annual report and investor presentation
3. Extract key transaction information manually
4. Prepare internal briefing notes
5. Discuss findings with Associate / VP
6. Build valuation and presentation materials

---

M&A Analyst Copilot Workflow

1. Upload public transaction documents
2. Extract transaction information into a structured format
3. Identify:
   - Acquirer
   - Target
   - Deal value
   - Purchase consideration (cash / stock / debt)
   - Strategic rationale
   - Expected synergies
   - Regulatory approvals
   - Key transaction risks
4. Generate follow-up due diligence questions for the analyst.
5. Produce an analyst-ready briefing note for further financial analysis.

The objective is not to replace analyst judgement.

Instead, the tool accelerates repetitive information gathering so analysts can focus on valuation, strategic analysis and client work.
